# Frame the job (Jobs To Be Done)

Frame the initiative as the **job the user is hiring it for**, not the feature you'd build.
*"People don't want a quarter-inch drill; they want a quarter-inch hole."*

## How
- State the **outcome** the user wants, in *their* words — the progress they're trying to make.
- Name the **context** (when/where the job arises) and **today's alternative** (what they do now, and
  what it costs them).
- Measure the **outcome**, not the output. "Feature shipped" is not the job done.

JTBD is the anchor that keeps you on outcomes; run it alongside `run-a-rat` — RAT first to test the
killer belief, JTBD to frame the work.
