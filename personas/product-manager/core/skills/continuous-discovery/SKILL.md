# Continuous discovery — the Opportunity Solution Tree

Don't jump to the first solution. Make the path from a desired outcome to a tested idea **visual and
shared**, widen the space, then narrow it. (Teresa Torres, *Continuous Discovery Habits*.)

## The Opportunity Solution Tree (OST)
```
desired OUTCOME  (one measurable goal)
  └─ OPPORTUNITIES  (unmet needs / pains / desires from real users)
       └─ SOLUTIONS  (≥3 options per opportunity)
            └─ ASSUMPTION TESTS  (the cheapest thing that could falsify each)
```
Build it top-down; it keeps everyone honest about *which* opportunity a solution serves and exposes the
opportunities you have no solution for.

## The habits
- **Interview about their life, not your idea** (the Mom Test): specific *past* behavior, not future
  hypotheticals. Hunt for the struggle; compliments are noise.
- **Weekly touchpoints** with real users by the people building — small, continuous, not a quarterly
  research phase. Decisions get fresh evidence, not stale.
- **Keep problems distinct from solutions** — never let a solution masquerade as a need; place it on the
  tree where it actually belongs.
- **Generate ≥3 solution options** per opportunity before choosing; the first idea is rarely the best.
- **Then converge:** `run-a-rat` tests the riskiest belief, `prioritize-tradeoffs` picks. Discovery
  widens; those two narrow.
- **Discovery is continuous, not a phase** — pair each build with assumption tests, not just delivery.
