# Account for distribution — a build must earn its way to users

The graveyard is full of good products nobody found. Distribution is a **product input**, not a
post-launch afterthought you hand to marketing. You don't *execute* GTM (sales/marketer do) — but the
product decision is incomplete until you can say how it reaches, activates, and retains people.

## The method
1. **Name the channel.** How will the *right* users actually encounter this — existing surface, search,
   integration/marketplace, referral, content, sales motion? "We'll tell people" is not a channel. If
   there's no plausible channel, that's a finding *before* the build, not after.
2. **Name the activation moment.** The first-use point where the user gets the value (the "aha"). Design
   the path to it; a feature whose value is buried behind setup won't land however good it is.
3. **Name the retention loop.** Why do they come back — a habit, a trigger, accumulating value, a
   network that grows? A build with no loop is a one-time hit, not a product.
4. **Sanity-check the economics.** Roughly: is the value delivered worth more than the cost to acquire +
   serve? A loop that costs more than it returns is a leak you're scaling.
5. **Consult, then own the decision.** Pull in sales/marketer for how to *run* the channel; you still
   own whether the product, as scoped, can be distributed at all. Bake the activation/loop requirements
   into the spec.

## The discipline
- Beware **local maxima** — a feature that delights the few who find it but has no path to the many.
- Distribution often *is* the product: the wedge, the integration, the shareable artifact. Treat
  reach as a design constraint, not a marketing problem to solve later.
